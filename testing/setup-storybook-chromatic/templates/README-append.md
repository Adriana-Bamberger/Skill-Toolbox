## 🎨 UI Component Workspace & Testing Workspace

This project leverages **Storybook v10+** for component development, **Vitest Browser Mode** for functional interaction testing, and **Chromatic** for cross-browser visual regression testing.

### 1. Launching Storybook Locally

To start the isolated design workspace environment and view your components locally, run:

```bash
  npm run storybook
```

Open http://localhost:6006 to interact with your component sandbox, test accessibility parameters, and run mock events.

### 2. Running Local Interaction Tests

To run your automated browser interaction specifications, query simulations, and assertion states via the terminal:

```Bash
  npm run test-storybook
```

### 3. One-Time Chromatic Setup (Visual Regression Cloud)

To activate cloud-based visual review workflows and cross-browser snapshot tracking:

1. Go to Chromatic.com and sign up for a free account using your GitHub/Git provider login.
2. Create a new project instance tracking this repository.
3. Copy your unique secure Project Token generated by the Chromatic dashboard.
4. Execute your initial pipeline upload using your token:

```Bash
  npx chromatic --project-token="YOUR_CHROMATIC_PROJECT_TOKEN"
```

5. Moving forward, you can run `npm run chromatic` locally or via GitHub Actions by populating the secret variable key storage location `$CHROMATIC_PROJECT_TOKEN`. TurboSnap is pre-configured to only analyze altered components, maximizing your monthly free-tier usage rules!
